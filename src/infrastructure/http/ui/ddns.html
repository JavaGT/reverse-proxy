Network → local addresses

Each row is one IP on a network interface (from the OS). Loopback is only for this machine; LAN/private addresses are how other devices on your network reach this host.

ColumnMeaning
InterfaceOS name (e.g. en0, eth0, lo, Docker bridges).
FamilyIPv4 or IPv6 for that address.
Internalyes = loopback (127.0.0.1 / ::1). no = addresses used on a real network (including private ranges like 192.168.x.x).
AddressThe IP string for that interface and family.

Public IP (Internet)

IPv4/IPv6 as seen by public HTTPS “what is my IP” services (egress). This is separate from the local table.

DNS resolution

The Network DNS table lists each configured apex and a catch-all probe per apex (*.apex with the real probe hostname under it; see the table footnote). Route hostnames appear only in Routes. Matches public marks when a resolved address matches the reported public IP.

Primary badge on IP addresses: In Network (DNS IPv4/IPv6 cells, local addresses, and public IP), an IP shown with the primary (highlighted) badge is one this server reports—either a local interface address or the current public IP from the lookup—so that badge means the value resolves or corresponds to this host.

Optional: keep your Porkbun DNS A and AAAA records aligned with this server’s public IP on a schedule. This does not obtain TLS certificates or register proxy routes.
